# Domain Language: JobTracker

## Core Concepts

| Term | Definition |
| :--- | :--- |
| **JD Scraper** | The automated system that extracts raw text from job listing URLs. |
| **Browser JD Capture** | A Chrome extension intake path that captures one or more user-selected job description sections from the current webpage and sends them into the Add Job review flow. |
| **Add Selection** | The extension action that appends the user's current page text selection to Capture Review. |
| **Capture Review** | The extension-side checklist where a user reviews, removes, and reorders selected job description sections before sending them to the app. |
| **JD Handoff** | A server-side record that transfers selected job description sections into Add Job and expires after 30 minutes or once consumed. |
| **Web App Session** | The signed-in app session that authorizes Browser JD Capture and JD Handoff creation. |
| **AI Extraction** | The process of converting raw job description text into structured data (company, title, skills, etc.). |
| **Match Score** | A percentage (0-100%) calculated by AI that estimates how well a user's master profile aligns with a specific job description. |
| **Master Profile** | The comprehensive, single source of truth for a user's professional data (experience, skills, education) used to seed all AI generations. |
| **Tailored CV** | A job-specific resume generated by AI that highlights the most relevant parts of the Master Profile for a specific JD. |
| **CV Builder Studio** | A dedicated workspace where a user can generate, edit, reorder, and hide Tailored CV sections, switch layouts, preview PDF output, and download the result. |
| **Professional ATS** | The default universal CV layout: single-column, ATS-first, professional, and minimal styling with standard section headings. |
| **CV Version Snapshot** | A saved CV state that stores user-edited CV content independently from Master Profile data for a specific use case (general or application-specific). |
| **Application Pipeline** | The visual workflow (Saved → Applied → Interview → etc.) that tracks the lifecycle of a job hunt. |

## Design Language

| Term | Definition |
| :--- | :--- |
| **Professional Futurism** | The visual style combining tech-forward elements (dark mode, glassmorphism) with professional clarity and high-trust aesthetics. |
| **Landing Page Promise** | Turn any job listing into a tracked application with an AI-tailored CV in minutes. |
| **Primary Landing Audience** | Solo job seekers applying to many roles who know generic CVs are reducing their chances. |
| **Hero Workflow Scene** | A product-style first-viewport visual showing JD URL capture, AI Extraction, Match Score, Tailored CV preview, and Application Pipeline status together. |
| **Grounded Landing Copy** | Landing page language that emphasizes saving jobs, understanding fit, tailoring CVs, and tracking next steps without exaggerated success claims. |
| **Product Preview Landing** | A landing page approach that sells the app through realistic interface scenes rather than abstract brand storytelling. |
| **Landing CTA Section** | A closing section that invites the user to start by turning their first job listing into a tracked application and Tailored CV. |
| **Landing Footer** | A compact footer with brand, product links, account links, social icons, and trust-oriented legal links. |
| **Primary Landing CTA** | The main call-to-action label is “Get Started”. |
| **Landing Social Icons** | The footer social icon set is LinkedIn, X, and Instagram. |

## Relationships

- **Browser JD Capture** creates a **JD Handoff** for the signed-in user.
- **Add Selection** appends one selected page section to **Capture Review**.
- **Capture Review** happens inside **Browser JD Capture** before a **JD Handoff** is created.
- **Browser JD Capture** relies on the user's **Web App Session** and does not store app auth tokens inside the extension.
- A **JD Handoff** feeds the existing Add Job review flow before **AI Extraction**.
- A **JD Handoff** is consumed when the Add Job review flow loads it and otherwise expires after 30 minutes.
- **Browser JD Capture** never creates an **Application** directly.
- **Browser JD Capture** complements the **JD Scraper** and manual paste paths; it does not replace them.
- A **Tailored CV** can be revised in **CV Builder Studio** before download.
- **Professional ATS** is the default user-facing CV template in CV Builder Studio.
- **CV Builder Studio** starts from Master Profile data and supports user edits per **CV Version Snapshot** without mutating **Master Profile**.
- Section order, entry order, and visibility are controlled per **CV Version Snapshot**.
- AI tailoring in CV Builder Studio rewrites content but does not automatically hide sections or entries.
- AI tailoring must not fabricate facts or metrics that are not present in user data.

## Flagged Ambiguities

- "Browser extension" is in scope only as **Browser JD Capture**: a narrow Chrome extension path for one or more user-selected JD sections and the current page URL, not fully automatic capture from any site.
